================================================================================
AKAR MASALAH UTAMA:
================================================================================

1. FITUR FRIENDS BELUM DIIMPLEMENTASIKAN
   - Tidak ada endpoint /api/friends/* di Next.js
   - Tidak ada friends_repository.dart di Flutter
   - Tabel friends/friend_requests mungkin ada tapi tidak ada API

2. PROFILE TIDAK DI-FETCH DARI SERVER
   - loadProfileFromStorage() hanya baca local cache
   - Perubahan di web tidak ter-sync ke mobile
   - Solusi: Tambah call ke /api/auth/mobile-session

3. GHOST MODE KEMUNGKINAN AKTIF
   - Jika is_ghost_mode = true di database
   - fetchNearbyUsers() akan skip dan peta kosong
   - Solusi: Cek dan set is_ghost_mode = false

4. ERROR TIDAK DITAMPILKAN DI UI
   - mapError dan chatError ada tapi tidak di-render
   - User tidak tahu ada masalah
   - Solusi: Tambah SnackBar untuk error

5. HEARTBEAT CHAT BERGANTUNG PADA MAP
   - Jika map error, chat tidak auto-refresh
   - Solusi: Pisahkan timer untuk chat

================================================================================
SOLUSI CEPAT (BISA DILAKUKAN SEKARANG):
================================================================================

1. CEK GHOST MODE DI SUPABASE (5 menit)
   ```sql
   SELECT id, username, is_ghost_mode FROM profiles WHERE id = 'USER_ID';
   -- Jika true:
   UPDATE profiles SET is_ghost_mode = false WHERE id = 'USER_ID';
   ```

2. JALANKAN RLS SETUP (5 menit)
   - Buka Supabase SQL Editor
   - Paste isi file supabase_rls_setup.sql
   - Run

3. TEST API MANUAL (10 menit)
   ```bash
   # Test map endpoint:
   curl "https://zmayy.vercel.app/api/map/visible?lat=-6.2&lng=106.8" \
     -H "Authorization: Bearer YOUR_TOKEN"
   
   # Test chat endpoint:
   curl "https://zmayy.vercel.app/api/chat/history" \
     -H "Authorization: Bearer YOUR_TOKEN"
   ```
