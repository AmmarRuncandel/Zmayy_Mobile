================================================================================
BAGIAN 1: PERBANDINGAN ARSITEKTUR
================================================================================

1.1 FLUTTER MOBILE
------------------
Arsitektur: Clean Architecture (Repository Pattern)

LAYER:
- Presentation: lib/features/* (screens, panels)
- Domain: lib/data/models/* (data models)
- Data: lib/data/repositories/* (API calls)
- Core: lib/core/* (config, theme, state management)

STATE MANAGEMENT: Provider (ChangeNotifier)
- ZmayyAppState: Global app state
- Repositories: Injected via Provider

ROUTING: MaterialApp + Navigator
- AppNavigator.navigatorKey untuk global navigation
- Manual navigation dengan Navigator.push/pop

1.2 NEXT.JS WEB
---------------
Arsitektur: App Router (Next.js 13+)

LAYER:
- UI: app/components/* (React components)
- API: app/api/* (route handlers)
- Hooks: app/hooks/* (custom React hooks)
- Context: app/context/* (React Context)
- Utils: app/utils/* (helper functions)

STATE MANAGEMENT: React Context + Hooks
- useAuth, useProfile, useChat (custom hooks)
- Context providers untuk global state

ROUTING: File-based routing (App Router)
- app/page.tsx → /
- app/login/page.tsx → /login
- app/api/*/route.ts → API endpoints

2.1 AUTENTIKASI
---------------

FLUTTER:
--------
FILE: lib/features/auth/login_screen.dart
FILE: lib/features/auth/register_screen.dart
FILE: lib/data/repositories/auth_repository.dart

ENDPOINT YANG DIGUNAKAN:
- POST /api/auth/mobile-login
- POST /api/auth/mobile-register
- GET /api/auth/mobile-session

FLOW:
1. User input email + password
2. AuthRepository.login() → API call
3. Response: { access_token, user, profile }
4. Save token ke SecureStorage
5. Navigate ke AppShell

NEXT.JS:
--------
FILE: app/login/page.tsx
FILE: app/api/auth/mobile-login/route.ts
FILE: app/api/auth/mobile-register/route.ts
FILE: app/api/auth/mobile-session/route.ts

FLOW:
1. User input email + password
2. supabase.auth.signInWithPassword()
3. Set cookie session
4. Redirect ke / (home)

PERBEDAAN:
----------
✅ Flutter: Token-based (Bearer token di header)
✅ Next.js: Cookie-based session (untuk web) + token API (untuk mobile)
✅ Flutter: Manual token management
✅ Next.js: Automatic session management via middleware

GAP:
----
❌ Flutter tidak punya endpoint untuk refresh token
❌ Flutter tidak punya endpoint untuk forgot password
❌ Flutter tidak punya endpoint untuk change password

2.2 PETA (MAP)
--------------

FLUTTER:
--------
FILE: lib/features/map/map_screen.dart
FILE: lib/data/repositories/map_repository.dart

ENDPOINT YANG DIGUNAKAN:
- GET /api/map/visible?lat=X&lng=Y
- POST /api/map/visible

LIBRARY: flutter_map, geolocator

FLOW:
1. Request location permission
2. Get current position
3. Fetch visible users dari API
4. Render markers di peta
5. Update setiap 30 detik (heartbeat)

NEXT.JS:
--------
FILE: app/components/map/MapViewInner.tsx
FILE: app/api/map/visible/route.ts

LIBRARY: react-leaflet

FLOW:
1. navigator.geolocation.watchPosition()
2. Update profiles.last_lat, last_lng setiap position change
3. Fetch visible users via RPC get_visible_users
4. Render markers dengan discriminator friend/stranger
5. Real-time update via Supabase realtime (optional)

PERBEDAAN:
----------
✅ Next.js: Continuous GPS tracking (watchPosition)
❌ Flutter: One-time position + periodic refresh (30s)
✅ Next.js: Automatic coordinate update ke database
❌ Flutter: Manual call enableMap()
✅ Next.js: Ghost mode wipe coordinates
❌ Flutter: Ghost mode hanya skip fetch

GAP:
----
❌ Flutter tidak punya continuous GPS tracking
❌ Flutter tidak auto-update coordinates ke database
❌ Flutter tidak punya ghost mode toggle di UI
❌ Flutter tidak punya recenter animation yang smooth

REKOMENDASI:
------------
1. Ganti Geolocator.getCurrentPosition() dengan getPositionStream()
2. Auto-update coordinates setiap 5-10 detik
3. Tambah ghost mode toggle di ProfilePanel
4. Tambah smooth animation untuk recenter

BAGIAN 7: PERBANDINGAN KEAMANAN
================================================================================

7.1 FLUTTER SECURITY
--------------------

TOKEN STORAGE:
- flutter_secure_storage untuk token
- Encrypted storage di device

AUTHENTICATION:
- Bearer token di Authorization header
- Manual token management
- No automatic refresh

API CALLS:
- ApiClient dengan timeout 15s
- Manual error handling
- No retry logic

7.2 NEXT.JS SECURITY
--------------------

SESSION MANAGEMENT:
- Cookie-based session untuk web
- Token-based untuk mobile API
- Automatic session refresh

MIDDLEWARE:
- proxy.ts untuk auth verification
- x-user-id injection
- Header sanitization

RLS:
- Supabase Row Level Security
- Policy-based access control
- Automatic enforcement

7.3 PERBEDAAN
-------------

✅ Flutter: Secure storage (encrypted)
✅ Next.js: Cookie (httpOnly, secure)

✅ Flutter: Manual token management
✅ Next.js: Automatic session management

✅ Flutter: Client-side validation only
✅ Next.js: Server-side + RLS validation

GAP:
----
❌ Flutter tidak punya automatic token refresh
❌ Flutter tidak punya retry logic
❌ Flutter tidak punya request queue
❌ Flutter tidak punya offline mode

REKOMENDASI:
------------
1. Implementasi automatic token refresh
2. Tambah retry logic untuk failed requests
3. Tambah request queue untuk offline mode
4. Tambah biometric authentication
